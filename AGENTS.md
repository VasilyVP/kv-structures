# AGENTS

This repository is a TypeScript monorepo for storage-engine-backed JS data structures.

## Start Here

- Read [README.md](README.md) for package overview.
- Read [packages/redis/README.md](packages/redis/README.md) for Redis usage and API examples.
- Read [packages/core/StructuredMap.ts](packages/core/StructuredMap.ts) and [packages/core/StructuredSet.ts](packages/core/StructuredSet.ts) before changing implementations.

## Repository Layout

- `packages/core`: Abstract contracts for async map/set behavior.
- `packages/redis`: Redis implementation (`RedisMap`, `RedisSet`, client lifecycle).
- `src/index.ts`: Root package exports.

Do not add Redis-specific logic to `packages/core`.

## Build And Test

Run from repository root:

- `npm install`
- `npm test`
- `npm run build`

What these do:

- Root `npm test` runs Vitest.
- Root `npm run build` runs [build.sh](build.sh), which builds `packages/redis`, then root package via `pkgroll`.

## Redis Runtime Requirements

- Tests require Redis running on `localhost:6379` unless custom client options are provided.
- Call `createClient()` before using `RedisMap`/`RedisSet`.
- Call `closeClient()` in teardown for tests or app shutdown.
- `createClient()` is singleton-style and returns existing client if already initialized.

## Code Conventions

- Use async APIs consistently; map/set operations are Promise-based.
- Preserve key naming conventions used in implementation:
  - map keys: `${mapName}:${key}`
  - autogenerated names: `kv-map-...`, `kv-set-...`
- Favor cursor iteration (`SCAN`/`SSCAN`) and batching for large datasets; avoid loading all keys at once.
- Keep ESM + CJS outputs intact in package exports (see [package.json](package.json) and [packages/redis/package.json](packages/redis/package.json)).

## Testing Notes

- Prefer isolated names per test and cleanup with `clear()` or `closeClient()`.
- Keep TTL-sensitive tests tolerant to timing jitter.
- If adding behavior to `RedisMap` or `RedisSet`, update tests in [packages/redis/tests](packages/redis/tests).
